# EDEN Asset Pipeline

Production assets are assembled by `npm run assets:sync`; it downloads the recorded source set, optimizes it, generates EDEN-owned ambient layers, and writes `public/assets/manifest.json` with file hashes and provenance. Run `npm run assets:verify` after refreshing files.

## Build Pipeline

`scripts/assets/sources.json` is the reviewed source ledger. The sync script:

1. Fetches the six GLB ecosystem models and selects Poly Haven 1K sources.
2. Compresses models using Draco and transcodes embedded material textures to WebP capped at 1024px.
3. Selects a 1K HDRI, transcodes standalone textures to WebP, and crops product imagery to optimized 1200 x 1500 JPEGs.
4. Encodes opt-in mono AAC ambient layers and copies local Draco decoder binaries needed by the browser.

Do not add downloadable production assets outside `sources.json` and the generated-asset section of `sync-assets.mjs`; this keeps license review and deployments reproducible.
